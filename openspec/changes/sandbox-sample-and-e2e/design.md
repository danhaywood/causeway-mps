## Context

`reference-app` already proved the idiom compiles and that generated-style and hand-written code
coexist on one classpath. This change makes that real *through the DSL*: a sandbox program generated by
the MPS generator, compiled against the same hand-written app. The headless build is the runner.

## Goals / Non-Goals

**Goals:** sandbox stubs resolve; a `customers` sample DSL program; generate → diff vs golden → compile
with the hand-written app on one classpath; the action body referencing `OrderService` compiles.

**Non-Goals:** the generator itself (separate change); the structure/type concepts (separate changes);
full runtime/UI verification beyond a deferred introspection smoke check.

## Decisions

**Stubs mirror the `reference-app` dependency set.** `causeway.sandbox` imports Causeway applib 3.6.0 +
Jakarta Persistence 3.1.0 + Jakarta Inject 2.0.1 + the `reference-app` jar, so embedded bodies resolve
`OrderService` and Causeway types. Rationale: identical classpath to the compile-verified golden.

**The sample mirrors the golden exactly.** module `customers`, entity `Customer` (`String name`,
`placeOrder(Product, int)` whose body calls `orderService.placeOrder(...)`), entity `Product`. So
generate→diff against `reference-app/.../customers/` is a precise pass/fail.

**One shared classpath, one-way generation.** Generated sources compile together with the hand-written
`app/OrderService` — proving the coexistence boundary holds in practice, not just in `reference-app`.

**Runtime introspection is a separate, deferred smoke check.** Compile-time coexistence is the milestone
here; booting a Causeway context to confirm `ENCAPSULATION_ENABLED` introspection is a follow-up.

## Risks / Trade-offs

- **Headless stub wiring may differ from the IDE's** → capture the exact sandbox library setup; align with
  `reference-app`; iterate against the build.
- **Generated output may not byte-match the golden** (formatting/imports) → compare semantically; accept
  benign formatting differences, investigate structural ones.
- **The embedded body's external reference might not survive generation** → this is the milestone; if it
  fails, it points back at `dsl-type-system`/generator, not this change.
